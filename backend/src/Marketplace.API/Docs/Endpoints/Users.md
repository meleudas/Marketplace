# UsersController — `/users`

Усі маршрути: **`[Authorize]`** (будь-який автентифікований користувач), окрім явно зазначених.

## `GET /users/me`

- **Призначення:** профіль маркетплейсу поточного користувача та список компаній з активним членством (`companyMemberships`: id, назва, slug, роль у компанії, `isOwner`).
- **Повертає:** **200** `UserDto`.
- **Авторизація:** Bearer.
- **Помилки:** **401** якщо немає user id у claims.
- **Примітки:** компанії з soft-delete не потрапляють у `companyMemberships`. У відповідях `GET /users` та `GET /users/search` поле `companyMemberships` завжди порожній масив.

## `GET /users`

- **Призначення:** список усіх користувачів з репозиторію.
- **Повертає:** **200** масив `UserDto`.
- **Авторизація:** Bearer (у коді **немає** обмеження роллю — фактично будь-хто з токеном).
- **Side effects:** немає (read-only).
- **Примітки:** з точки зору продукту варто вважати це **чутливим** endpoint-ом; для адміністрування краще додати `Admin` policy на рівні контролера/політики.

## `GET /users/search?userName=...`

- **Призначення:** пошук за іменем (частковий матч).
- **Приймає:** query `userName` (обов’язковий для логіки handler — порожній дасть помилку).
- **Повертає:** **200** масив `UserDto`.
- **Авторизація:** Bearer.
- **Помилки:** failure якщо `userName` порожній (текст від handler).

## `DELETE /users/{id}`

- **Призначення:** м’яке видалення **власного** акаунта.
- **Приймає:** path `id` (guid) — має збігатися з поточним користувачем.
- **Повертає:** **200** порожнє тіло.
- **Авторизація:** Bearer.
- **Side effects:** soft-delete профілю/акаунта (логіка `IUserManagementService`).
- **Помилки:** **403** якщо `id` ≠ поточний user; **401** без user id.

## `PATCH /users/{id}/role`

- **Призначення:** призначити **глобальну** роль маркетплейсу (`UserRole`: `Buyer`, `Seller`, `Moderator`, `Admin` — у body як рядок, case-insensitive).
- **Приймає:** path `id`; body `{ "role": "buyer|seller|moderator|admin" }` (`AssignUserRoleRequest`).
- **Повертає:** **200** порожнє тіло при успіху (`Result` без значення).
- **Авторизація:** **`[Authorize(Roles = "Admin")]`** + Bearer.
- **Side effects:** оновлення ролі в Identity/порті автентифікації.
- **Помилки:** **400** невалідна `role`; інші — з `IAuthenticationPort`.
