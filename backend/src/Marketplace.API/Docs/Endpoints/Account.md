# AccountController — `/account`

## `POST /account/confirm-email`

- **Summary (1 рядок):** Підтвердження email за токеном з листа.
- **Призначення:** підтвердження email (Identity + доменний `User.Verify()`).
- **Хто може викликати:**
  - JWT: не потрібна
  - Глобальні ролі: —
  - Компанійні ролі: —
- **Бізнес-логіка:**
  1. Валідувати `email` та `token`
  2. Підтвердити email в Identity (`EmailConfirmed`)
  3. Оновити `isVerified` у профілі маркетплейсу
- **Side effects (синхронно):** `EmailConfirmed`, `isVerified` у профілі маркетплейсу
- **Async / «магія»:** —
- **Де на фронті:**
  - Екран: Confirm email page
  - API-модуль: `frontend/src/features/auth/api/auth.api.ts` (`confirmEmail`)
  - Статус: `implemented`
- **Приймає:** body `ConfirmEmailRequest` (`email`, `token`).
- **Повертає:** **200** порожнє тіло при успіху.
- **Помилки:** невалідний/прострочений токен.

  2. Зчитати стан 2FA з `IAuthenticationPort`
  3. Повернути `TwoFactorStatusDto`
- **Side effects (синхронно):** —
- **Async / «магія»:** —
- **Де на фронті:**
  - Екран: Security settings (planned)
  - API-модуль: —
  - Статус: `planned`
- **Приймає:** немає.
- **Повертає:** **200** `TwoFactorStatusDto` (структура з `IAuthenticationPort`).
- **Помилки:** **401** без user id.
